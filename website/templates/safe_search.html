Почему это безопасно?

Попробуйте те же SQL-инъекции, что работали на уязвимой странице:

  • ' OR '1'='1 - найдет только пользователей с таким текстом в имени
  • ' UNION SELECT id, username, password, email FROM users-- - будет искать как строку
  • '; DROP TABLE users;-- - не выполнит команду DROP

Как работает защита:

  • Пользовательский ввод передается как параметры, а не как часть SQL-запроса
  • База данных четко разделяет данные и команды
  • Специальные символы (', ;, --) обрабатываются как данные, а не как команды SQL

Код на Python:


# Уязвимый код (НЕПРАВИЛЬНО):
sql = f"SELECT * FROM users WHERE username = '{username}'"

# Безопасный код (ПРАВИЛЬНО):
sql = "SELECT * FROM users WHERE username = ?"
cursor.execute(sql, (username,))
