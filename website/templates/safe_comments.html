Добавить комментарий (безопасная версия)

Внимание: Форма отправляет данные на уязвимый endpoint. Для полной безопасности нужна и безопасная форма тоже.

Комментарии (с экранированием HTML)

{% if messages %}
{% for msg in messages %}
{{ msg.username }} {{ msg.timestamp }}
{{ msg.content }}
Исходный текст: {{ msg.content }}
{% endfor %}
{% else %}

Пока нет комментариев.

{% endif %}

Почему это безопасно?

Попробуйте те же XSS-атаки, что работали на уязвимой странице:

  • <script>alert('XSS')</script> - отобразится как текст, а не выполнится
  • <img src="x" onerror="alert(1)"> - будет показан как текст
  • <iframe src="javascript:alert('XSS')"></iframe> - не создаст iframe

Как работает защита:

  • Специальные HTML-символы заменяются на сущности:
    • < → &lt;
    • > → &gt;
    • & → &amp;
    • " → &quot;
    • ' → &#x27;
  • Браузер отображает эти символы как текст, а не интерпретирует как HTML

Код на Python:


# Уязвимый код (НЕПРАВИЛЬНО):
# В шаблоне: {{ content|safe }}

# Безопасный код (ПРАВИЛЬНО):
# В представлении:
from html import escape
safe_content = escape(user_content)

# В шаблоне:
{{ content }}  # автоматически экранируется

Дополнительные меры защиты:

  • Content Security Policy (CSP) - ограничивает источники скриптов
  • HTTPOnly cookies - предотвращают кражу cookies через JavaScript
  • Валидация ввода на стороне сервера
  • Библиотеки для очистки HTML (bleach, html-sanitizer)
